We do not store any personal information about you other than the name of your choice.

Privacy

Data that can be lead to a specific person is stored using hashing algorithms.

This means that nobody but you and optionally your peers may connect the data within certain containers.

For example
You invite your friend Carl to a plan using his Facebook-account. When he signs in using his Facebook-account we use his facebook identity (like facebook.com/users/123456789) and convert it into a irreversible userKey (might be something like: bf74e4a280affafbdf6692bc6a9f3d66b03094fbfb4a91589bc7fd6b32664fdb)

Data we collect

We collect and use the minimal set of information to be able to runt the service.

This means we store common text and information such as plan names and task titeles etc in plain text under the hashed container for a plan. For users we currently store the username only in the "collaborators" list for the adminstrators eyes only. The exposed set will use the display name which origin is agnostic to the system. IE a colaborator added from an admins facebook-list will use the unique id hashed and the users display name.

How we store data

The current demo utilizes a component called local storage.

This means that the data is stored in a browser built-in database. To clear all data stored by you, simply clear the cache in your browser.
